SEO basics and an intro to <meta> tags
SEO basics and an intro to <meta> tags
24/01/24・Written by Pieter Edwards

SEO is an abbreviation commonly used in web development, it stands for Search Engine Optimization and is the process of providing enough information so web browsers can show your website to the right people. Good SEO is crucial for any and all websites as without it you will not reach your audience and your efforts will be wasted. To provide the browser with this information you must enlist the help of the <meta> tag.

All <meta> tags are located within the <head> section of your document. This simple tag has a whole host of properties that you can use to your advantage in the SEO game. When a Search Engine (such as Google) indexes your website it will look for all the <meta> tags and use them in order to decide when to display your site in search results and, yes, whilst a lot of what determines your sites position in the results is out of your control (being based on factors such as site popularity and your clickthrough rate) using good SEO techniques is a good foot in the door and a firm foundation to build success from.

So, first things first, the <meta> tags you will likely already be familiar with are:

<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">

These two tags are often already within your document when you create a new HTML project as Visual Studio Code includes them for you but what do they mean? The “ charset=’UTF-8’ ” tells the browser that you are using Unicode Transformation Format 8 which is used to display nearly all characters within the Unicode consortium. The second <meta> tag controls the viewport as it both sets the width of the viewport to the user's device but also ensures the page does not load zoomed in so all elements maintain their original size. Both these tags are useful for accessibility reasons but not so much SEO. The main <meta> tag you want to use has the attribute “keywords”

<meta name="keywords" content="clock, open, source">

This tag will give the Browser more keywords to look for and the more of these words that appear in a user’s search query, the more likely it is for your site to be displayed in the list of results. To add this to your project:

  • Create a new <meta> tag
  • Set the name attribute to “key words”
  • In the content field write each individual word separated by commas
<meta name="description" content="An example description!">

This tag provides the browser with the little paragraph that it will display below the link to your site. Creating a short but informative description is key to entice users onto your page as this will be their only way of knowing what your site is about if they’re a first time user.

<title>Does jQuery have a place on the modern web?</title>

The <title> tag is also important as this gives the name of your page and is used as the text that shows in the Search results. These words will also be used for matching with search queries so make sure they’re relevant and helpful to the user as this is public facing unlike the keywords tag!

So that was a beginners guide to Search Engine Optimization and the use of <meta> tags, there are more tags than I have spoken about in this article but they’re more advanced and go into specifying and controlling how Search Engines index your website. For now these tags will allow you to begin to optimise your project and see more users. I do still have two final things to mention however. To be able to see your pages Search Analytics make sure to go to Google Search Console and add your site to your account and once you’ve proved ownership you can view data in interactive graphs that will allow you to see your best performing searches and much more! The final useful software is the Chrome browsers inbuilt Lighthouse test. Simply right click on your website, press “Inspect” and then navigate the tabs to the “Lighthouse” section. This will test your site in many different areas but we’re mainly interested in the SEO category as it gives you a score and tells you how to improve it. I hope this article was helpful and will enable you to do better in your web dev career!
